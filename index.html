Split Slider Demo

Enable Auto-SlideDisable Auto-Slide

This demo of Split Slider was initialized with the following script:

mySlider = $("#split-slider-demo").splitSlider({
	leftArrow: 'images/arrow_left.jpg',
	rightArrow: 'images/arrow_right.jpg'
});

The simplest version of the slider can be defined with this call:

$("#split-slider-demo").splitSlider();

This plugin was developed on jQuery 1.7.1.

A complete list of all available options follows:

  • Defining Your Elements: (Expand)
    • Number of Elements - The total number of elements to be displayed at a time. This includes the "middle" element, if used, but not the arrows.
      (Expected: Integer) 
      numVisible: 5
    • Show a "Middle" Element - If you want a static "middle" element, set this to true. Otherwise, set to false.
      (Expected: Boolean) 
      showMiddle: true
    • Middle Slide Element - The element identifier used to find the pre-defined "middle" element. (Currently, this must be a child of the element you call the plugin on.)
      (Expected: String) 
      middleDiv: '.middle-slide'
    • List of Slide Elements - The identifier of the container element which holds all of the slides as its direct children. (Tested with a <ul> list, but it should work with <div>s as well.)
      (Expected: String) 
      slideList: '.slides'
  • Slide Styles: (Expand)
    • Element Width - The width of the output <div> elements. This is an inline style provided by the plugin for each ".slide" element.
      (Unit of Measure: px, Expected: Integer) 
      width: 168
    • Element Height - The height of the output <div> elements. This is an inline style provided by the plugin for each ".slide" element.
      (Unit of Measure: px, Expected: Integer) 
      height: 220
    • Border Width - The total width of left and right side borders on each ".slide" element. No borders are provided by the plugin, but this value will help take into account any borders you provide in your styles. (The default stylesheet provided with this plugin includes a 1px transparent border around all ".slide" elements, and a 1px dotted border between elements [which replaces the transparent border on that side].)
      (Unit of Measure: px, Expected: Integer) 
      borderWidth: 2
  • Navigation Arrows: (Expand)
    • Left Arrow Image - The relative URL to the image you wish to use for the Left Arrow. (This is placed directly into an image tag, so it should be relative to the HTML page, not the plugin directory.)
      (Expected: String) 
      leftArrow: 'arrow_left.png'
    • Right Arrow Image - The relative URL to the image you wish to use for the Right Arrow. (This is placed directly into an image tag, so it should be relative to the HTML page, not the plugin directory.)
      (Expected: String) 
      rightArrow: 'arrow_right.png'
  • Animation Effects: (Expand)
    • Duration - The amount of time it takes for the element to slide out or back in. (By default, it takes 1200ms to slide out, and another 1200ms to slide back in.
      (Unit of Measure: ms, Expected: Integer) 
      duration: 1200
    • Easing - Easing function to use for the animation. (Currently, this has not been tested to see if it works with jQuery UI Easing, but there's no real reason it shouldn't.)
      (Expected: String) 
      slideEasing: 'linear'
    • Animation Overflow - The slides will animate this many pixels outside the container, allowing a small window of time that no slides are visible during the animation time.
      (Unit of Measure: px, Expected: Integer) 
      slideOutOverflow: 40
    • Auto Slide - Should the slider automatically trigger the slide animation periodically? (The auto slide functionality will be paused if a user hovers over the slider container.)
      (Expected: Boolean) 
      autoSlide: false
    • Auto Slide Wait Time - The amount of time after the slide animation stops before the next begins, when Auto Slide is active.
      (Unit of Measure: ms, Expected: Integer) 
      autoSlideTimeout: 4000

And, finally, an explanation of some of the features included in this plugin! (And some extra tricks!)

  • Features: (Expand)
    • Static Middle Element (optional)

      The initial purpose of the script was to have a central element that stayed in one place, with an even number of items on either side which slide outward and off the page, to be replaced with the next set of items.

    • Even or Odd Amount of Elements

      The tasks handled by the plugin soon grew to include being able to move all elements (half, rounded up, on the left; half, rounded down, on the right).

      With this in place, we further allowed an even number of elements to be displayed. In this case, half of the items go in each direction.

    • Auto-Slide

      Because, after all, what jQuery Slider plugin is ever complete without automatic sliding as an option?!

    • Looking to the Future....

      Future iterations of this plugin may include features such as the ability to slide up/down instead of left/right and more! Please feel free to suggest some features!

  • Extras: (Expand)
    • Programmatically Slide Left/Right

      If you have defined the Slider as in the first example, you can now reference mySlider with a limited selection of functions. This includes the ability to programmatically "click" the left and right arrows.

      mySlider.slideLeft();		// Simulates clicking the "Left" arrow.
mySlider.slideRight();		// Simulates clicking the "Right" arrow.
    • Enabling Auto-Slide (after page load)

      When we were looking to add the Auto-Slide function to this plugin, we determined that you should not only be able to slide left or right programmatically, but you should also be able to enable and disable Auto-Slide. This uses the slider variable defined when calling the Slider initially, as above.

      	// These both start Auto-Slide, if it's not activate already.
mySlider.autoSlide();
mySlider.autoSlide(true);

	// And this turns it off.
mySlider.autoSlide(false);
